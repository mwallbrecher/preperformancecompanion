## Feature 1 – Sensor-Based Balancing Game (Attention Reset)

Type: Physical + attentional grounding  
Duration: ~30–60 seconds  

### Concept
A short, intuitive balancing game where the user tilts their phone to guide a ball through a simple labyrinth.

Inspired by physical maze toys (see reference image).

### Purpose
- interrupt rumination loops
- shift attention from internal → external
- engage body + focus simultaneously
- create fast cognitive reset

### Core Interaction
- user tilts phone (using gyroscope / accelerometer)
- ball moves accordingly
- simple goal: reach exit OR just keep balance

### UX Principles
- no instructions required
- instantly understandable
- no failure state needed (optional)
- no scoring, no gamification pressure

### Flow
1. Instant entry (no setup)
2. Ball appears + starts moving
3. User tilts phone to control movement
4. Optional:
   - reach goal OR
   - run for fixed time (e.g. 30 sec)
5. auto exit or manual exit

### Constraints
- max session: 60 seconds
- minimal UI
- no text-heavy screens
- no complex level design (start simple)

## Feature 2 – Thought Offloading (Voice → Node Mapping)

Type: Cognitive offloading + distraction  
Duration: ~30–90 seconds  

### Concept
User speaks their thoughts into the phone.  
The system captures and organises them into a simple node-based visual structure (inspired by Obsidian-style graphs).

### Purpose
- externalise overthinking
- reduce mental load
- create sense of “captured thoughts”
- allow user to move on

### Core Interaction
- tap to start speaking
- voice recording
- real-time or delayed visualisation
- thoughts appear as nodes

### UX Principles
- no typing required
- minimal friction to start
- no need to organise manually
- visually calming, not complex

### Flow
1. Entry screen (calm prompt)
   → “Say what’s on your mind”
2. Tap to start recording
3. User speaks freely
4. System generates nodes (live or after)
5. Nodes gently expand / organise
6. Exit

### Constraints
- no heavy editing features
- no tagging system initially
- no deep navigation
- focus on *release*, not management

### Optional Enhancements (later)
- cluster nodes by topic
- fade-out animation on exit
- simple keyword highlighting

## Feature 3 – Time-Based Focus Fill (Passive Stabilisation)

Type: Passive grounding / focus stabilisation  
Duration: user-defined (e.g. 30s – 3 min)

### Concept
A minimal screen that gradually fills with a fluid/liquid-like animation over time.

The user does nothing except stay with the screen.

### Purpose
- reduce stimulation
- create calm temporal anchor
- encourage stillness
- support focus before performance

### Core Interaction
- user starts session
- screen slowly fills (like water/liquid)
- no interaction required

### UX Principles
- zero cognitive load
- passive experience
- calming, not boring
- no decisions required

### Flow
1. Entry
2. Start
3. Liquid fills over time
4. Optional:
   - subtle breathing rhythm sync
5. End state (full screen)
6. Exit or restart

### Controls
- Reset button
- Exit button

### Constraints
- no text during session
- no distractions
- smooth animation is key
